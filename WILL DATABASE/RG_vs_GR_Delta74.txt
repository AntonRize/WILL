\subsection{Geometric Incompatibility of Phase Parameterization}
\hypertarget{sec:gr-incompatibility}{}\label{sec:gr-incompatibility}

General Relativity cannot structurally integrate the relational phase $O$ as the fundamental dynamical parameter without collapsing its pseudo-Riemannian axioms. The empirical advantage of the phase-coupled orbital geometry cannot be subsumed by the standard geodesic equation via coordinate transformation.

In a pseudo-Riemannian manifold, the invariant interval defines the affine parameter $\tau$:
\begin{equation}
d\tau^2 = -\frac{1}{c^2} g_{\mu\nu} dx^\mu dx^\nu \label{eq:invariant-interval}
\end{equation}
Orbital dynamics are strictly governed by the affine geodesic equation:
\begin{equation}
\frac{d^2 x^\mu}{d\tau^2} + \Gamma^\mu_{\alpha\beta} \frac{dx^\alpha}{d\tau} \frac{dx^\beta}{d\tau} = 0 \label{eq:affine-geodesic}
\end{equation}
Introduce a monotonically increasing relational phase parameter $O = O(\tau)$. Applying the chain rule to the derivatives yields the non-affine transformation:
\begin{equation}
\frac{dx^\mu}{d\tau} = \frac{dx^\mu}{dO} \frac{dO}{d\tau} \label{eq:chain-velocity}
\end{equation}
\begin{equation}
\frac{d^2x^\mu}{d\tau^2} = \left(\frac{dO}{d\tau}\right)^2 \frac{d^2x^\mu}{dO^2} + \frac{d^2O}{d\tau^2} \frac{dx^\mu}{dO} \label{eq:chain-acceleration}
\end{equation}
Substituting (\ref{eq:chain-velocity}) and (\ref{eq:chain-acceleration}) into (\ref{eq:affine-geodesic}) and isolating the second derivative with respect to $O$ produces the non-affine geodesic equation:
\begin{equation}
\frac{d^2x^\mu}{dO^2} + \Gamma^\mu_{\alpha\beta} \frac{dx^\alpha}{dO} \frac{dx^\beta}{dO} = -f(O) \frac{dx^\mu}{dO} \label{eq:non-affine-geodesic}
\end{equation}
where the parameterization constraint $f(O)$ is defined as:
\begin{equation}
f(O) = \frac{d^2O/d\tau^2}{(dO/d\tau)^2} \label{eq:f-constraint}
\end{equation}
The term $-f(O) (dx^\mu/dO)$ is strictly proportional to the tangent vector. The spatial geometry of the invariant spacetime trajectory remains unaltered. The affine geometric form (\ref{eq:affine-geodesic}) is conserved exclusively if $f(O) = 0$, necessitating:
\begin{equation}
O = a\tau + b \label{eq:affine-condition}
\end{equation}
The relational phase $O$ is dynamically generated from the local ratio of dimensionless projections ($\kappa$, $\beta$). Consequently, $dO/d\tau$ is non-constant, dictating $f(O) \neq 0$.

This establishes a strict mathematical dichotomy:
1. If $O$ functions purely as a non-affine coordinate relabeling (diffeomorphism), the physical curve in spacetime remains identical to the standard General Relativity trajectory. An identical geometric curve produces an identical empirical output.
2. If $O$ operates as an independent dynamic generator that modifies the physical trajectory to yield the observed orbital closure, the trajectory is no longer the solution to (\ref{eq:affine-geodesic}). 

Severing the equations of motion from the affine parameter $\tau$ decouples the trajectory from the metric interval (\ref{eq:invariant-interval}). The Christoffel symbols $\Gamma^\mu_{\alpha\beta}$ lose their differential grounding. While the First Post-Newtonian (1PN) approximation can mimic the parameter $O$ through algebraic curve-fitting upon a fixed background, full General Relativity cannot natively execute phase-driven mechanics without discarding the pseudo-Riemannian manifold. In simple words GR would have to become RG to fully adopt this method.

def integrate_orbit(params, pad_days=40.0, rtol=1e-10, method='DOP853'):
    a_as, e, i, om0, Om, P, T0, V0, K = params
    a_km = (K * P * DAY * np.sqrt(1 - e**2)) / (2 * np.pi * np.sin(i))
    n = 2 * np.pi / (P * DAY)
    GM = n**2 * a_km**3
    c2 = C_KMS**2
    rp = a_km * (1 - e)
    vp = math.sqrt(GM * (1 + e) / rp)
    s0 = [rp * math.cos(om0), rp * math.sin(om0),
          -vp * math.sin(om0), vp * math.cos(om0)]

    def rhs(t, s):
        x, y, vx, vy = s
        r2 = x * x + y * y
        r = math.sqrt(r2)
        v2 = vx * vx + vy * vy
        rdot = (x * vx + y * vy) / r
        A = -GM / (r2 * r) * (1.0 + v2 / c2 - 4.0 * GM / (c2 * r))
        B = 4.0 * GM * rdot / (c2 * r2)
        return [vx, vy, A * x + B * vx, A * y + B * vy]

    tmin = (min(t_a.min(), t_r.min()) - T0 - pad_days) * DAY
    tmax = (max(t_a.max(), t_r.max()) - T0 + pad_days) * DAY
    sol_b = solve_ivp(rhs, (0.0, tmin), s0, method=method,
                      rtol=rtol, atol=[1e-3, 1e-3, 1e-9, 1e-9], dense_output=True)
    sol_f = solve_ivp(rhs, (0.0, tmax), s0, method=method,
                      rtol=rtol, atol=[1e-3, 1e-3, 1e-9, 1e-9], dense_output=True)

    def state_at(mjd):
        ts = (np.atleast_1d(mjd) - T0) * DAY
        out = np.empty((4, ts.size))
        m = ts < 0
        if m.any():   out[:, m] = sol_b.sol(ts[m])
        if (~m).any(): out[:, ~m] = sol_f.sol(ts[~m])
        return out
    return state_at, GM, a_km

def model_integrated(params, mjd_a, mjd_r, state_pack=None):
    a_as, e, i, om0, Om, P, T0, V0, K = params
    state_at, GM, a_km = state_pack if state_pack else integrate_orbit(params)

The hybrid model uses \(\omega \propto \nu\), i.e. the pericentre advance is driven by the true anomaly – the orbital phase – rather than by coordinate time. It achieves \(\chi^2 = 727.6\), matching WILL.

You could claim this is “just a GR model with a different parametrisation.” But that misses the point. **In GR, the true anomaly is a derived function of time.** You solve Kepler’s equation to get \(\nu(t)\), then you apply \(\omega(\nu)\). The fundamental independent variable is still coordinate time \(t\). The fact that you evaluate \(\omega\) via \(\nu\) rather than directly via \(t\) is a mathematical convenience; the underlying ontological commitment – that time is the external parameter – is unchanged.

But here is the deeper problem for GR:

**Why does the phase‑coupled parametrisation work so much better than the time‑linear one?**

If time is truly an external, linearly‑flowing coordinate, then a model with time‑linear precession (\(\omega = \omega_0 + \dot\omega\, \Delta t\)) should be the natural, physically motivated choice. The fact that it performs *drastically worse* – and that tying precession to the orbital phase closes the entire \(\Delta\chi^2 \approx 74\) gap – tells you something about the structure of the world:

**The data prefer a description in which precession is governed by the accumulated orbital phase, not by an external clock.**

That is exactly what WILL predicts from first principles: the precession law is \(\Delta\phi = 2\pi \tau_Y^2/(1-e^2)\), where \(\tau_Y\) is a function of the relational phase projections. The phase accumulation – not coordinate time – determines the dynamics.

GR has no explanation for why precession should couple to orbital phase rather than time. The phase‑coupled parametrisation is *ad hoc* in GR; it is inserted by hand because it fits better. In WILL, it is derived. That is the difference between a descriptive model and a generative theory.
